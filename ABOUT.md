# About dev-plan-assistant

## 项目简介

**dev-plan-assistant** 是一个全生命周期开发助手 skill，专为 AI 辅助编程设计。它的核心理念是「**先想清楚，再做干净，最后记得复盘**」，覆盖规划、开发、回顾三个阶段的完整开发闭环。

## 核心价值

这个 skill 不是帮你写代码，而是帮你**避免写出不需要的代码**：

- **开发前**：想清楚要做什么、有没有现成方案
- **开发中**：模块进度有迹可循、不遗漏不重复
- **开发后**：Bug 可追溯、经验可沉淀

## 设计哲学

### 1. 避免重复造轮子

在动手写代码之前，通过多层次 GitHub 技术调研确认没有现成的开源方案：
- 搜索类似开源项目（awesome-selfhosted, openalternative.co, alternativeto.net）
- 搜索工具库和框架（npm, PyPI, GitHub topics）
- 六维评估体系：Activity / Adoption / Security / License / Docs / Fit

### 2. 从实际代码中学习

扫描项目现有代码，提取已有的模式做成模板文档。这不是代码规范，而是从实际代码中提取的「项目已经怎么做的」快照。

### 3. 模块化思维

采用架构师视角的功能域划分，一个模块包含前后端全链路，覆盖完整业务流程，而非拆分成零散的小模块。

## 三阶段生命周期

```
┌────────────── 阶段一：规划 ──────────────┐
│  1.1 反问理解需求                         │
│  1.2 GitHub 技术调研（避免重复造轮子）    │
│  1.3 扫描项目生成模板                     │
│  1.4 制定开发计划 + 生成 VuePress 站点    │
├────────────── 阶段二：开发 ──────────────┤
│  2.1 模块启动 → 创建模块开发计划          │
│  2.2 模块更新 → 更新进度状态              │
│  2.3 模块完成 → 标记 completed            │
├────────────── 阶段三：回顾 ──────────────┤
│  3.1 Bug 文档（按需）                     │
│  3.2 测试文档（按需）                     │
│  3.3 总结文档（按需）                     │
└──────────────────────────────────────────┘
```

## 适用场景

- **新项目启动**：从需求理解到技术选型，避免盲目开发
- **功能扩展**：在现有项目中添加新模块
- **代码重构**：梳理现有代码，制定重构计划
- **团队协作**：标准化的文档结构，便于知识传承

## 跨平台兼容

支持多个 AI 编程平台：
- Claude Code
- WorkBuddy
- Trae
- 其他支持 skill 机制的 AI 助手

## 文档结构

项目生成 VuePress 站点，结构如下：

```
docs/
├── 01-需求分析/      # 需求理解输出
├── 02-技术调研/      # GitHub 调研结果
├── 03-模板中心/      # 从现有代码提取的模式模板
├── 04-开发计划/      # 模块开发计划（一个模块一个文档）
├── 05-开发规范/      # 编码/Git 规范
└── 06-项目跟踪/      # Bug/测试/总结文档
```

## 目标用户

- 追求高效开发的独立开发者
- 需要规范化流程的团队
- 希望减少重复劳动的 AI 辅助编程用户
- 重视文档和知识沉淀的技术团队
