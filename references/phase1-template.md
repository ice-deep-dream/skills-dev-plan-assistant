# 阶段一·1.3：扫描项目生成模板

> 在制定开发计划之前，先扫描项目现有代码，提取已有模式做成模板文档。
> 这不是代码规范，而是「项目已经怎么做的」快照。新模块开发时直接参照，避免一人一个写法。

## 核心理念

**模板服务于项目** — 从现有代码中提取，不是凭空定义。如果项目还没有代码（全新项目），跳过这一步。

## 执行时机

- **老项目加新功能**：必须扫描，确保新模块风格一致
- **全新项目**：跳过，没有代码可扫
- **项目刚初始化**：项目骨架已有但模块不多时，建议跑一次

## 扫描步骤

### Step 1：判断项目类型

读取 `package.json` 或项目根目录，判断项目类型：
- 前端：有 `react` / `vue` / `angular` / `vite` / `webpack`
- 后端：有 `express` / `nestjs` / `koa` / `fastapi` / `gin`
- 全栈：同时包含前后端目录

### Step 2：遍历关键目录

按项目类型扫描关键目录：

```
前端项目重点扫描：
  src/api/          → API 调用方式
  src/store/        → 状态管理
  src/router/       → 路由配置
  src/components/   → 通用组件
  src/utils/        → 工具函数

后端项目重点扫描：
  src/routes/       → 接口定义
  src/middleware/   → 中间件
  src/models/       → 数据模型
  src/config/       → 全局配置
  src/utils/        → 工具函数
```

### Step 3：提取模式

对每个扫描项：
1. 找 2-3 个典型文件作为样本
2. 提取**核心结构**（去掉业务逻辑，保留骨架）
3. 标注关键调用链和依赖关系

### Step 4：写入模板文档

每个模板文档使用以下格式：

```markdown
---
title: 模板名称
type: template
category: frontend | backend
created: YYYY-MM-DD
---

# 模板名称

## 使用场景
<!-- 什么时候用这个模板 -->

## 典型代码
<!-- 从现有代码中提取的骨架，标注 ✂️ 表示可替换部分 -->

## 关键依赖
<!-- 用了哪些全局模块 -->

## 注意事项
<!-- 容易踩的坑 -->
```

---

## 输出位置

所有模板写入 `docs/03-模板中心/`，更新 `README.md` 作为索引。

---

## 使用方式（后续开发时）

开发新模块时：
1. 先看 `03-模板中心/README.md` 了解有哪些模板
2. 找到对应的模板参照写法
3. 如果发现模板与实际不符，更新模板（模板服务于项目）

---

## 示例：扫描结果

假设扫描一个 Vue + Pinia + Axios 项目，生成：

```
docs/03-模板中心/
├── README.md                    # 模板索引
├── 前端-API调用.md               # axios 实例封装、拦截器、错误处理
├── 前端-Store结构.md             # Pinia store 定义模式
├── 前端-路由模式.md              # Vue Router 配置、守卫
├── 前端-通用组件.md              # 被多处引用的组件
├── 后端-接口模式.md              # Express 路由 + controller
├── 后端-中间件.md                # 鉴权、日志中间件
├── 后端-数据库操作.md            # Prisma 模型操作模式
└── 项目-目录结构.md              # 整体目录说明
```
